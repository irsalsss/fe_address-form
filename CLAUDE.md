## Folder structure

```
src/
  features/
    address-form/
      api/          TanStack Query hooks, fetchers (e.g. POST /addresses)
      hooks/        feature-local hooks (Google Places, country switching)
      stores/       feature-local Zustand stores (form draft, UI state)
      schemas/      Zod schemas per country + shared validation
      __tests__/    co-located unit + integration tests
      types.ts      Country, Address union, AddressFormData
      index.ts      public surface — only file other features may import
  shared/
    lib/            cn() utility, framework-agnostic helpers
    hooks/          cross-feature hooks
    types/          shared types
    config/         zod-validated env.ts (VITE_API_URL, VITE_GOOGLE_PLACES_API_KEY)
    api/            TanStack Query client + HTTP wrapper
    i18n/
      index.ts      i18next init (en default, id secondary)
      locales/<locale>/<ns>.json
      resources.d.ts  strict t() key typing
  app/              providers (QueryClient), router (createBrowserRouter)
  components/
    ui/             shadcn/ui components (added via npx shadcn@latest add)
public/
tests/
  setup.ts          @testing-library/jest-dom/vitest import
e2e/
  *.spec.ts         Playwright specs (chromium + firefox + webkit)
```

### Import rules (enforced by ESLint boundaries)

- `features/<a>` may not import from `features/<b>` internals — use `features/<b>/index.ts`
- `shared/` may not import from `features/`
- `components/` may not import from `features/`
- Absolute imports use `@/` alias (resolves to `src/`)

## State management

| State kind | Tool | Where |
|---|---|---|
| Server state (fetched addresses, country lists) | @tanstack/react-query | `features/address-form/api/` |
| Client UI state (manual-edit toggle, draft form) | zustand | `features/address-form/stores/` |
| Form state + validation | react-hook-form + zod via @hookform/resolvers/zod | schemas in `features/address-form/schemas/` |

**Rules:**
- Never store server data in Zustand. Server state has cache, retry, invalidation.
- Never store pure UI toggles in TanStack Query. Wrong tool, wrong lifecycle.
- One store per feature, max. If a feature needs multiple stores, split the feature first.
- Every form goes through React Hook Form with a Zod resolver. No ad-hoc `useState` validation.
- The Zod schema is the single source of truth for both client validation and server input parsing.

## Routing

All routes in `src/app/router.tsx`. Lazy-load route components via `React.lazy` once features have UI.

## Testing

### Unit + Integration (Vitest + Testing Library)
- **Vitest only.** No Jest. No Karma.
- Testing Library (`render`, `screen`, `userEvent`) for component tests.
- No snapshot tests — they rot. Use explicit assertions.
- Co-located: `foo.ts` → `foo.test.ts` (or `__tests__/foo.test.ts`).
- Integration tests live in `tests/` at the package root.

### E2E (Playwright)
- Specs in `e2e/`.
- Three browser projects: chromium, firefox, webkit.
- Dev server auto-started by `webServer` config (port 3001).
- `pnpm test:e2e` for headless, `pnpm test:e2e:ui` for UI mode.

## Tailwind v4 + shadcn/ui

- Tailwind v4: CSS-first config (`@import "tailwindcss"` in `src/index.css`).
  No `tailwind.config.ts`. The `@tailwindcss/vite` plugin handles processing.
- shadcn/ui CSS variables scaffolded (neutral theme, light + dark).
  These are shadcn defaults — **theme customization belongs to `frontend-design`.**
- Add components via: `npx shadcn@latest add button` (outputs to `src/components/ui/`).
- The `cn()` helper in `src/shared/lib/utils.ts` merges Tailwind classes via `clsx` + `tailwind-merge`.

## Localization (i18n)

- Locales: `en` (default), `id` — fallback `en`
- Messages: `src/shared/i18n/locales/<locale>/<namespace>.json`
- Default namespace: `common`
- i18next initialized in `src/shared/i18n/index.ts`, imported in `main.tsx` before `<App />`
- Strict typing via `resources.d.ts` — missing keys break type-check
- **Adding a key:** add to `en/common.json` first, then `id/common.json`.
  CI / type-check enforces parity.
- Feature namespaces: create `src/shared/i18n/locales/<locale>/address-form.json`

## Conventions

- **File naming:** `kebab-case.ts` for utilities, `PascalCase.tsx` for components, `useThing.ts` for hooks.
- **Barrel exports:** `index.ts` only at feature roots — not deep. Deep barrels kill tree-shaking.
- **Import order:** external → workspace → `@/` absolute → `./` relative.
- **TS strictness:** `strict: true`, `noUncheckedIndexedAccess: true`.
  Never `any`. `unknown` + narrowing instead.
- **Env:** only read via `src/shared/config/env.ts` (zod-validated).
  Never `import.meta.env` elsewhere.

## Adding a new feature

1. `mkdir -p src/features/<name>/{api,hooks,stores,schemas,__tests__}`
2. Create `types.ts` and `index.ts` (public surface)
3. Add TanStack Query hooks in `api/` — never call `fetch` outside `shared/api/` and feature `api/`
4. Add Zustand store in `stores/` if needed (one max per feature)
5. Add Zod schemas in `schemas/`
6. Mount the feature's route in `src/app/router.tsx`
7. Write tests co-located in `__tests__/`
8. Add locale strings in `src/shared/i18n/locales/<locale>/<feature>.json`
